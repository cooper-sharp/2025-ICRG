\subsection{Overview}\label{overview}

This notebook revises the original \texttt{code/analysis/logit.Rmd}
workflow to better match the current analytic plan.

\subsubsection{Primary decisions}\label{primary-decisions}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Use a \textbf{hierarchical logistic regression} to predict \textbf{PG
  vs HC}.
\item
  Keep \textbf{moderators in Block 4} only.
\item
  Treat \textbf{DOSPERT as domain-specific} in the primary model.
\item
  \textbf{Exclude the gambling domain} from the primary model because of
  likely criterion overlap with PG status.
\item
  Treat \textbf{investment/financial risk} cautiously. By default, it is
  excluded from the primary model, but a switch is included below in
  case you want to add it.
\item
  Run two follow-up models:

  \begin{itemize}
  \tightlist
  \item
    a \textbf{non-gambling DOSPERT composite} sensitivity model,
  \item
    an \textbf{exploratory gambling-domain} model.
  \end{itemize}
\item
  Run a \textbf{within-PG severity analysis} predicting GSAS total.
\end{enumerate}
